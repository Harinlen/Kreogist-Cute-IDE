Building your program

This section will show you how to compile your source code in Cuties and execute the binary.

Compile your source code

In Cuites, you don't need to compile your source code, link, or build execution manually. We provide three function to help you compile and build execution automatically:

  • Compile and Run
  • Compile
  • Run

Compile and Run

Compile and Run command can compile your file via the compiler which you set or provided by default, and launch kciConsoleRunner to execute the binary. For more information, see Compile and Run.

Compile

To compile your program, you have to check your compile configure. For more information, see Compiler Configure.
Select Main Menu, Execute, Compile. Cuties will show compile dock and begin compile process. In the compile dock, the compile infos will display on the text panel. It will be something like this:

22:32:24 Preparing Compiler.
22:32:24 Current Compiler Details:
g++ 4.2.1
22:32:24 Compile Output:
/usr/bin/gcc /Users/Saki/helloworld.c -lm -ggdb -Wall -o /Users/Saki/helloworld
22:32:24 Compile Successful.

Cuties will read your configure and try to get the compiler details. And then try to compile the file via compiler with the configure. All the compiler output will be shown in the text panel. Those infos will be arranged and displayed in the tree view panel as well. You can get more information about the error like the position of the error occur by clicking one error line of the tree view panel. Double click one line will jump to the line error occur.

Run

Before you run your program, make sure your program has been compiled. If not, Cuties will mentioned you to compile your program first. Unlike other IDEs, Cuties provides you an awesome binary executor, "kciConsoleRunner". It will show you the totally running time of your program.
