这些版本间的区别

带有 MinGW 和 FPC 编译器的 Cuties：提供了一个带有 MinGW 编译器和 FPC 编译器的 Cuties。将使用 MinGW 套件中的 GDB 作为调试器。
带有 TDM-GCC 和 FPC 编译器的 Cuties： 提供了一个带有 TDM-GCC MinGW 编译器和 FPC 编译器的 Cuties。将使用 TDM-GCC MinGW 中附带的 GDB 作为调试器。
带有 MinGW 编译器的 Cuties： 提供了一个仅带有 MinGW 编译器的 Cuties。将使用 MinGW 套件中的 GDB 作为调试器。
带有 TDM-GCC 编译器的 Cuties： 提供了一个仅带有 TDM-GCC MinGW 编译器的 Cuties。将使用 TDM-GCC MinGW 套件中的 GDB 作为调试器。
带有 FPC 编译器的 Cuties： 提供了一个仅带有 FPC 编译器的 Cuties。这个版本不支持调试。
没有编译器的 Cuties：只提供一个 Cuties。你需要下载自己的编译器并且重新配置Cuties。


关于 MinGW 和 TDM-GCC:

MinGW 是 “Minimalist GNU for Windows”的缩写，是为开发原生Microsoft Windows应用程序的一个最小的开发环境。开发者可以用于原生Microsoft Windows平台上的开发，也可以用于交叉编译。MinGW包括如下的组件：

  • 一个为GNU Compiler Collection（GCC）提供的接口，包括 C、C++、ADA 和 Fortran 编译器；
  • 适用于 Windows 的 GNU Binutils （汇编语言、连接器、档案管理器）；
  • 一个用于 Windows 系统的 MinGW 和 MSYS 命令行安装工具（mingw-get）；
  • 一个图形化命令行安装工具的封装（mingw-get-inst）；

GNU Compiler Collection 包括了为 C、C++、Objective-C、Fortran、Java、Ada 以及 Go 语言提供的前端支持，以及这些语言的库（libstdc++、libgcj……）。GCC 最初是为 GNU 操作系统开发的编译器。GNU 操作系统是被设计为一个完全的自由软件，尽力保证用户的权力自由。

TDM-GCC 是一个 Windows 下的编译器套装。它包含了绝大部分稳定版本的 GCC 工具集，一部分更加亲和于 Windows 系统的补丁以及自由和开源的 MinGW 或 MinGW-w64 开发套件。

在 Cuties 中，我们提供了 TDM-GCC 编译器版本是为了获得更小的磁盘占用。如需获得更多有关 MinGW 的信息，请查看http://www.mingw.org/。如需获得更多有关 TDM-GCC 的信息，请查看http://tdm-gcc.tdragon.net/。
